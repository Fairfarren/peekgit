# 贡献指南

感谢你关注 `peekgit`。

本项目欢迎 Issue、文档改进、测试补充和代码贡献。为保证协作效率，请先阅读本指南。

## 提交前准备

1. Fork 并克隆仓库。
2. 基于仓库当前默认分支创建功能分支（当前为 `master`）。
3. 安装 Go 1.24 及以上版本。
4. 本地执行以下检查：

```bash
go test ./...
go vet ./...
```

## 本地开发

项目配置了 [Air](https://github.com/cosmtrek/air) 热重载工具，可在代码变更时自动重新编译运行：

```bash
# 安装 Air（如果尚未安装）
go install github.com/cosmtrek/air@latest

# 启动热重载开发模式
air
```

`.air.toml` 已配置好，无需额外设置。

## 开发约定

- 保持改动聚焦：一个 PR 尽量只解决一个问题。
- 优先补充或更新测试，避免引入回归。
- 不要提交与本次目标无关的重构或格式化噪音。
- 提交信息建议清晰描述“为什么改”。

## 提交 Pull Request

请在 PR 描述中包含：

1. 背景与问题
2. 解决方案
3. 影响范围
4. 验证方式（命令与结果）

如果是 UI 或交互变更，请补充截图或录屏说明。

## Issue 反馈建议

提交 Bug 时请尽量包含：

- 复现步骤
- 预期行为与实际行为
- 运行环境（操作系统、Go 版本）
- 日志或错误信息

## 行为规范

参与本项目即代表同意遵守 `CODE_OF_CONDUCT.md`。
