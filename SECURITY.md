# 安全策略

感谢你帮助提升 `peekgit` 的安全性。

## 支持范围

默认关注 `master` 分支上的最新代码。历史版本原则上不单独维护。

## 漏洞报告

请勿在公开 Issue 中直接披露可利用细节。

建议通过以下方式私下报告：

1. 在仓库中创建安全相关 Issue（避免贴出可利用细节）
2. 在描述中提供：
   - 漏洞类型与影响面
   - 复现步骤
   - 可能的修复建议

我们收到后会尽快确认并评估优先级。

## 处理流程

1. 维护者确认问题是否成立
2. 评估风险等级和影响范围
3. 制定并合并修复
4. 在必要时发布安全说明

## 协作原则

- 负责任披露
- 给维护者合理修复窗口
- 在修复发布前避免公开利用细节
